## üìã Developer Onboarding Checklist

When a new developer joins the project:

- [ ] Add developer information to this document
- [ ] Add developer to GitHub repository (invite as collaborator)
- [ ] Set appropriate branch permissions
- [ ] Provide access to development documentation
- [ ] Share project structure documentation
- [ ] Share coding standards (`.cursorrules`)
- [ ] Share version control guidelines (`docs/Services/github/VERSION_CONTROL.md`)
- [ ] Add developer to team communication channels
- [ ] Update `README.md` with developer information
- [ ] Update `.cursorrules` if needed

---

## üìã Developer Offboarding Checklist

When a developer leaves the project:

- [ ] Update status to "Inactive" in this document
- [ ] Remove GitHub repository access
- [ ] Revoke branch permissions
- [ ] Archive developer's contributions
- [ ] Update `README.md` if needed
- [ ] Transfer any critical knowledge/documentation
- [ ] Update contact information if needed
