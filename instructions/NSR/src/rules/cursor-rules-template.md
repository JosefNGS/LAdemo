**Purpose**: Template for generating `.cursorrules` files using NSR framework rules

**Version**: 1.0.0  
**Last Updated**: January 2026  
**Owner**: NorthStar Team

---

## Cursor Rules Structure

This template generates `.cursorrules` files that include:

1. **Pre-Flight Checklist** - Mandatory questions before making changes
2. **CHANGELOG Requirements** - Mandatory changelog updates
3. **Critical Rules** - Version control, task ownership, documentation
4. **Project Structure** - File organization rules
5. **Coding Standards** - TypeScript, React, styling guidelines
6. **Service Rules** - Service-specific requirements
7. **Development Workflows** - Task synchronization, auto-updates

## Key Sections

### 1. Pre-Flight Checklist
- Developer identity verification
- Change scope assessment
- Task ownership verification
- Documentation compliance
- Version control workflow
- Testing requirements
- Context understanding

### 2. CHANGELOG Mandatory Requirements
- Every change MUST be logged
- Developer/Cursor tracking required
- Folder-specific changelogs
- No exceptions policy

### 3. Critical Rules
- Main branch protection
- Pull Request workflow
- Task ownership enforcement
- Bidirectional auto-update requirements
- README/CHANGELOG in every folder

### 4. Project Structure Rules
- File organization standards
- Documentation location rules
- Service folder structure
- Developer profiles location

### 5. Coding Standards
- TypeScript strict typing
- React component patterns
- Tailwind CSS guidelines
- Naming conventions
- Code organization

## Generation Process

When generating `.cursorrules`:
1. Start with this template
2. Include relevant rules from `src/rules/` folder
3. Add project-specific rules
4. Include service rules if applicable
5. Add developer information
6. Include tech stack details
