# Instructions

## Overview
This folder contains external frameworks and instruction sets that guide development practices and methodologies.

## Purpose
The `instructions/` folder houses external frameworks and instruction sets that are referenced by the project but are not modified directly. These frameworks provide:
- Development methodologies
- Code standards and best practices
- Workflow guidelines
- Architecture patterns

## Folder Structure

### BMAD-METHOD
- **`BMAD-METHOD/`** - Business Model & Architecture Development Method framework
  - Complete framework source files
  - Documentation and guides
  - Sample modules
  - Development tools

### .agent-os
- **`.agent-os/`** - Agent Operating System configuration
  - Command definitions
  - Core instructions
  - Product documentation
  - Code standards
  - Security rules

### NSR
- **`NSR/`** - NorthStar Rules
  - Rules developed by the NorthStar team
  - Guidelines and principles for NorthStar projects

## Usage

### For Developers
- **Reference Only**: These frameworks are for reference - do not modify core framework files
- **Follow Guidelines**: Use framework guidelines when working on related features
- **Check Rules**: See `rules/` folder for extracted rules from these frameworks

### For AI Agents
- **Pre-Flight Checklist**: Reference framework documentation when needed
- **Compliance**: Follow framework guidelines when applicable
- **Rules Extraction**: Use `rules/` folder for quick reference

## Important Notes

### DO NOT Modify
- **Core Framework Files**: Do not modify files within `BMAD-METHOD/` or `.agent-os/` core directories
- **Framework Structure**: Do not reorganize framework internal structure
- **Source Files**: Framework source files should remain unchanged

### DO Use
- **Documentation**: Read and reference framework documentation
- **Samples**: Use sample modules as reference
- **Rules**: Use extracted rules in `rules/` folder for quick reference
